# Writing Docs

Documenting your package for users.

## Why this exists

Good documentation helps users understand and use your package effectively.

## How it works

Include a README.md with usage examples, API documentation, and installation instructions.

## Examples

````markdown
# My Package

Brief description of what the package does.

## Installation

```bash
kex install github:user/my-package
```
````

## Usage

```kx
use my-package { myFunction }

myFunction(arg)
```

## API

- `myFunction(arg)`: Description of what it does

```

## Common mistakes

- Not documenting function parameters
- Missing usage examples
- Outdated documentation
```

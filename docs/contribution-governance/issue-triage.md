# Issue Triage

Process for managing and prioritizing GitHub issues.

## Why this exists

Effective issue management helps focus development efforts on important problems.

## How it works

Issues are labeled, prioritized, and assigned based on impact and feasibility.

## Examples

Issue labels:

- bug: Software defects
- feature: New functionality
- documentation: Docs improvements
- good first issue: Beginner-friendly

## Common mistakes

- Not providing enough information in issues
- Duplicating existing issues
- Not using issue templates

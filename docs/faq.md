# FAQ

Frequently asked questions about Kexra.

## General

### What is Kexra?

Kexra is a simple, educational scripting language designed for learning programming concepts. It's built with TypeScript and runs on Node.js.

### Who is Kexra for?

- Beginners learning programming
- Students studying language design
- Developers exploring interpreted languages
- Educators teaching programming concepts

### Is Kexra production-ready?

No. Kexra is designed for learning and experimentation, not large-scale production applications.

## Technical

### Why is Kexra built on JavaScript?

JavaScript provides:
- Cross-platform compatibility
- Rich ecosystem
- Easy deployment
- Familiar syntax for many developers

### How does Kexra compare to other languages?

| Feature | Kexra | Python | JavaScript |
|---------|-------|--------|------------|
| Learning curve | Very easy | Easy | Medium |
| Performance | Slow | Fast | Fast |
| Ecosystem | Minimal | Huge | Huge |
| Use case | Learning | General | Web |

### Can Kexra run existing code?

No. Kexra has its own syntax and runtime. It's not compatible with other languages.

## Features

### Does Kexra support object-oriented programming?

Not yet. Kexra focuses on procedural programming with functions. OOP may be added in future versions.

### Can Kexra handle large programs?

Kexra is designed for small to medium programs. For large applications, consider more robust languages.

### Is there a package manager?

Not yet. Dependency management is planned for future versions.

## Future

### What's next for Kexra?

See the [roadmap](roadmap.md) for planned features including:
- Arrays and objects
- Standard library

- Better tooling

### Will Kexra become a "real" language?

The goal is educational value, not production dominance. Kexra will remain focused on learning and experimentation.

## Troubleshooting

### "Command not found" error

Ensure Kexra is installed globally:

```bash
npm install -g kexra
```

Check if npm global bin is in your PATH.

### Syntax errors

Review the [language basics](language-basics.md) and [errors](errors.md) documentation.

### Performance issues

Kexra is interpreted and not optimized for performance. For speed-critical applications, use compiled languages.

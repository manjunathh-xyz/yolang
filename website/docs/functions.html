Functions

Kexra supports user-defined functions with parameters and return values. Functions help organize code and make it reusable.

Defining Functions

Use the fn keyword to define a function:

fn add(a, b) {
  return a + b
}

Calling Functions

Call functions by name with parentheses:

set result = add(2, 3)
say result  # Outputs: 5

Function Scope

Functions have their own local scope. Variables defined inside functions are not accessible outside:

fn test() {
  set local = "inside"
  return local
}

say test()  # Outputs: inside
say local   # Error: undefined variable

However, functions can access global variables:

set global = "world"

fn greet() {
  return "Hello, " + global
}

say greet()  # Outputs: Hello, world

Return Values

Use return to exit a function and return a value:

fn multiply(x, y) {
  return x * y
}

say multiply(4, 5)  # Outputs: 20

If no return statement is used, the function returns null:

fn printMessage(msg) {
  say msg
  # No return statement
}

set result = printMessage("Hello")
say result  # Outputs: null

Parameters

Functions can have zero or more parameters:

fn greet() {
  say "Hello!"
}

fn addThree(a, b, c) {
  return a + b + c
}

greet()              # Outputs: Hello!
say addThree(1,2,3)  # Outputs: 6

Complete Example

fn factorial(n) {
  check n <= 1 {
    return 1
  } else {
    return n * factorial(n - 1)
  }
}

say factorial(5)  # Outputs: 120

Important Notes

  • Functions must be defined before they are called
  • Parameters are passed by value
  • Functions can call other functions
  • Recursion is supported
